Theory & Fundamentals

What is an Array?

An Array is a collection of elements stored in contiguous memory, each identified by an index. Because addresses are computed arithmetically, any element is reachable in O(1).

That random access is the array's superpower — and its cost: inserting or deleting in the middle forces every later element to shift, which is O(n).

Why Arrays matter. They're the most fundamental data structure: the backing store for strings, stacks, heaps, hash tables, dynamic arrays (vector), and matrices. Nearly every interview warm-up is an array problem.
Types of Array
▭
Static Array
Fixed size known at compile time. No resizing.
Use: fixed buffers, C-style int a[n].
↔️
Dynamic Array
Grows/shrinks at runtime; doubles capacity when full (amortized O(1) push).
Use: std::vector, the default choice.
▦
Multi-dimensional
Arrays of arrays — matrices, grids, tensors.
Use: DP tables, images, graphs as matrices.
🔤
Character Array
Contiguous chars; the basis for strings.
Use: strings, parsing, buffers.
🔢
Prefix / Difference
Derived arrays for O(1) range sums / range updates.
Use: range queries, subarray sums.
🧮
Sorted Array
Kept in order to enable binary search in O(log n).
Use: search, two pointers, merge.
Terminology
Index
Zero-based position of an element, a[0] is the first.
Contiguous memory
Elements stored back-to-back, enabling O(1) address math.
Subarray
A contiguous slice a[i..j].
Subsequence
Elements in order but not necessarily contiguous.
Capacity vs size
Dynamic arrays reserve capacity ≥ size to amortize growth.
Shift
Moving elements to make room (insert) or close a gap (delete).

Advantages & Disadvantages

Advantages

  • O(1) random access by index.
  • Excellent cache locality (contiguous memory).
  • Minimal per-element overhead.
  • Backbone of strings, stacks, heaps, matrices, and DP tables.

Disadvantages

  • Insert/delete in the middle is O(n) (shifting).
  • Static arrays have a fixed size.
  • Dynamic-array resizing occasionally copies everything.
  • Wasted capacity when over-reserved.

int main() {
    vector<int> a = {10,20,30,40,50};

    a.push_back(60);           // append — amortized O(1)

    sort(a.begin(), a.end());   // O(n log n)
    bool f = binary_search(a.begin(),a.end(),30); // O(log n)
    return 0;
}
patterns.cpp · Binary search, two pointers, window
// Binary search — sorted array, O(log n)
int bsearch(vector<int>& a, int t){
    int lo=0, hi=a.size()-1;
    while(lo<=hi){ int mid=lo+(hi-lo)/2;
        if(a[mid]==t) return mid;
        if(a[mid]<t) lo=mid+1; else hi=mid-1; }
    return -1;
}

// Two pointers — pair sum in sorted array, O(n)
bool pairSum(vector<int>& a, int t){
    int i=0, j=a.size()-1;
    while(i<j){ int s=a[i]+a[j];
        if(s==t) return true;
        s<t ? i++ : j--; }
    return false;
}

// Sliding window — max sum of k elements, O(n)
int maxSumK(vector<int>& a, int k){
    int sum=0; for(int i=0;i<k;i++) sum+=a[i];
    int best=sum;
    for(int i=k;i<a.size();i++){ sum+=a[i]-a[i-k]; best=max(best,sum); }
    return best;
}

Key Patterns
Two pointers. On a sorted array, move a pointer from each end inward to solve pair-sum, container-with-most-water, and dedup in O(n) — no extra space. Same-direction pointers handle in-place filtering and partitioning.
Sliding window & prefix sums. A moving window gives O(n) answers for "max/min subarray of size k" and variable-size condition problems. A prefix-sum array answers any range sum in O(1) after O(n) preprocessing.

Key Takeaways
⚡O(1) access, O(n) middle edits: contiguous memory buys instant indexing but makes insert/delete shift everything after.
🔍Sort unlocks log n: a sorted array enables binary search and two-pointer tricks — often worth the O(n log n) upfront.
↔️Two pointers & windows: the highest-yield array patterns, turning O(n²) brute force into O(n) with O(1) space.
🔢Prefix sums: precompute once, answer any range sum in O(1) — the base for many subarray problems.
