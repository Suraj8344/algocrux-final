Theory & Fundamentals

What is a Deque?

A Deque (double-ended queue, pronounced "deck") allows insertion and deletion at both ends — front and rear — in O(1). It generalizes both the stack and the queue.

Backed by a doubly linked list or a circular dynamic array (like C++ std::deque), giving O(1) at the ends and O(1) indexed access in the STL version.

Core operations

pushFront(x) / pushBack(x)
popFront()  / popBack()
front()     / back()
// all O(1)
Why Deques? A deque is a stack and a queue at once. It powers the sliding-window-maximum pattern, undo/redo with bounded history, palindrome checks, and work-stealing schedulers.
Deque as stack & queue
📚
As a Stack
Use only pushBack/popBack (or only the front) → pure LIFO.
Use: when you may later need the other end too.
🎟️
As a Queue
Push at back, pop at front → pure FIFO.
Use: BFS, buffering.
📈
Monotonic Deque
Maintain increasing/decreasing order by popping from the back before pushing.
Use: sliding window max/min in O(n).

Advantages & Disadvantages

Advantages

  • O(1) insert/remove at both ends.
  • Serves as both a stack and a queue.
  • std::deque adds O(1) random access.
  • Backbone of the sliding-window-maximum pattern.

Disadvantages

  • More overhead than a plain queue/stack.
  • std::deque memory is chunked, not fully contiguous.
  • Middle insertion/deletion is still O(n).
  • Iterator invalidation rules are subtle.
C++ STL Implementation
deque_stl.cpp · std::deque
#include <bits/stdc++.h>
using namespace std;

int main() {
    deque<int> dq = {20, 30};
    dq.push_front(10);   // [10,20,30]
    dq.push_back(40);    // [10,20,30,40]
    dq.pop_front();      // [20,30,40]
    dq.pop_back();       // [20,30]
    cout << dq.front() << " " << dq.back() << " " << dq[0];
    return 0;  // 20 30 20 — all O(1)
}
sliding_window_max.cpp · Monotonic deque
// Sliding Window Maximum — O(n)
vector<int> maxSlidingWindow(vector<int>& a, int k) {
    deque<int> dq;   // stores indices, values decreasing
    vector<int> res;
    for (int i = 0; i < a.size(); i++) {
        while (!dq.empty() && dq.front() <= i - k) dq.pop_front(); // stale
        while (!dq.empty() && a[dq.back()] < a[i]) dq.pop_back();  // smaller
        dq.push_back(i);
        if (i >= k - 1) res.push_back(a[dq.front()]); // front = max
    }
    return res;
}

Key Patterns
Sliding window maximum. Keep indices in a deque with values in decreasing order. Pop the back while it's smaller than the incoming value, pop the front when it slides out of the window. The front is always the window max — O(n) total.
Palindrome check. Compare front() and back(), pop both inward. If they ever differ, it's not a palindrome. O(n)/O(1).

Key Takeaways
↔️Both ends, O(1): a deque is stack + queue combined — push/pop/peek at front and back all constant time.
📈Monotonic deque: the signature use — sliding window max/min in O(n) by keeping values ordered.
🎯std::deque: the STL version adds O(1) random access via chunked storage, unlike list-backed deques.
🧭0-1 BFS: push front for 0-weight, back for 1-weight edges — Dijkstra-quality paths without a heap.
