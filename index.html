Every algorithm starts with a structure

Before sorting, searching, or backtracking - understand what you are operating on. Here is the lineup, roughly in the order most courses teach them.

A contiguous block of memory holding elements in order. Indexing is instant because the address of any element is just base + i times size - no searching required.

Tree

A hierarchy of nodes with one root and no cycles. Binary search trees keep smaller values left and larger right, turning search into a series of left/right decisions.

Graph

Nodes (vertices) connected by edges, with no fixed shape - trees, networks, and maps are all graphs. Traversed with BFS for shortest hops or DFS for full exploration.

BFS O(V+E)DFS O(V+E)Cyclic OK
09
🔑

Hash Table

A key is run through a hash function that maps it to a bucket index, giving near-instant lookup. Collisions (two keys, same bucket) are resolved with chaining or probing - the reason worst-case lookup can degrade to O(n).
