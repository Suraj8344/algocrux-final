Search a 2D Matrix I & II

LC
Practice both variants on LeetCode to reinforce your understanding.
Open Problem

Variant I: treat the matrix as one sorted list and binary search it. Variant II: walk a staircase from the top-right corner

What does it solve?

Search for a target in a matrix with sorted structure. Matrix I: each row is sorted and every row's first value exceeds the previous row's last, so the whole thing is one sorted sequence. Matrix II: rows and columns are each sorted, but rows don't chain together.

Key insight: for I, map a 1-D index idx to cell (idx/cols, idx%cols) and binary search over rows·cols → O(log(mn)). For II, start at the top-right: bigger than target → move left, smaller → move down. Each step drops a row or column → O(m+n).

The core idea

Complexity

Matrix I · time
O(log mn)

Binary search over all m·n cells using the flat-index trick. O(1) space.

Matrix II · time
O(m + n)

The staircase walk eliminates one row or column per step from the top-right. O(1) space.
