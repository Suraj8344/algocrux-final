Subarray Sum Equals K

LC

Open Problem

Sliding window fails with negatives, so track running prefix sums in a hash map and look up prefix − k in O(1) — O(N) time

What does it solve?

Count how many contiguous subarrays sum to exactly k. Because values can be negative, a sliding window won't work, so we lean on prefix sums instead.

Key insight: a subarray (j, i] sums to k exactly when prefix[i] − prefix[j] = k, i.e. prefix[j] = prefix[i] − k. Keep a hash map of prefix-sum → how many times it occurred. At each i, add the count of prefix − k. Seed the map with {0: 1} so subarrays starting at index 0 are counted. One pass → O(N).

The core idea

// count[prefixSum] = occurrences
count = {0: 1}; pre = 0; ans = 0
for x in nums:
  pre += x
  ans += count.get(pre − k, 0)
  count[pre] += 1

Complexity

Time complexity
O(N)

Single pass. Each element does one O(1) map lookup and one O(1) insert. No nested subarray scan.

Space complexity
O(N)

In the worst case every prefix sum is distinct and stored in the map.

C++ implementation

subarray_sum_k.cpp 
#include <bits/stdc++.h>
using namespace std;

// Count contiguous subarrays summing to k (works with negatives)
int subarraySum(vector<int>& nums, int k) {
    unordered_map<int,int> count;
    count[0] = 1;               // empty prefix: enables subarrays from index 0

    int pre = 0, ans = 0;
    for (int x : nums) {
        pre += x;                  // running prefix sum
        // subarrays ending here that sum to k = times we saw pre-k before
        if (count.count(pre - k)) ans += count[pre - k];
        count[pre]++;              // record this prefix sum
    }
    return ans;
}

int main() {
    vector<int> nums = {1,1,1};
    cout << subarraySum(nums, 2) << "\n"; // 2
    return 0;
}
