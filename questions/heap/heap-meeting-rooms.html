Meeting Rooms II — Greedy + Heap

Sort by start. A min-heap of room end-times tells you if the earliest-freeing room is open yet — O(N log N) time

The min-heap of end times

Given meeting intervals, how few rooms cover them all? The greedy plan: process meetings in order of start time. Keep a min-heap of the end times of rooms currently in use, so the room that frees up soonest is always on top.

For each meeting, peek the heap. If its earliest end time is ≤ this meeting's start, that room is free: reuse it (pop, push the new end). Otherwise every room is busy: open a new one (push). The answer is the largest the heap ever grows.

// sort meetings by start
for [s, e] in meetings:
  if heap && heap.top() <= s:
    heap.pop() # reuse a room
  heap.push(e)
answer = max(answer, heap.size())

int minMeetingRooms(vector<vector<int>>& meetings) {
    if (meetings.empty()) return 0;

    // 1) sort by start time
    sort(meetings.begin(), meetings.end());

    // 2) min-heap of end times of rooms in use
    priority_queue<int, vector<int>, greater<>> pq;

    for (auto& m : meetings) {
        int s = m[0], e = m[1];
        // earliest-freeing room is done ⇒ reuse it
        if (!pq.empty() && pq.top() <= s) pq.pop();
        pq.push(e);            // occupy a room until e
    }
    return pq.size();   // rooms still needed = peak overlap
}
