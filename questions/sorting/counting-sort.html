Counting Sort — Count, Accumulate, Place

Count occurrences of each value, compute prefix sums, then place elements in sorted order. O(N + K) time.

What does it solve?

Sorts integers in linear time by counting how many times each value appears. No comparisons between elements at all. Works best when the range of values (K) is not significantly larger than the number of elements (N).

Key insight: if you know an element's count and the count of everything smaller, you know its exact final position. Prefix sums of the count array give you that.

Three Phases

// Phase 3: Place elements (right to left)

Phase Description
1. CountScan input left to right. For each element, increment count[val - min].
2. Prefix SumAccumulate: count[i] += count[i-1]. Now count[i] = position after last occurrence of value i+min.
3. PlaceScan input right to left. Place each element at output[--count[val-min]]. This preserves stability.
4. DoneOutput array is the final sorted result.
C++ — Counting Sort (Stable) 
vector<int> countingSort(vector<int>& arr) {
    int n = arr.size();
    int mn = *min_element(arr.begin(), arr.end());
    int mx = *max_element(arr.begin(), arr.end());
    int range = mx - mn + 1;

    vector<int> count(range, 0), output(n);

    // Phase 1: Count occurrences
    for (int val : arr)
        count[val - mn]++;

    // Phase 2: Prefix sum
    for (int i = 1; i < range; i++)
        count[i] += count[i - 1];

    // Phase 3: Place elements (right to left for stability)
    for (int i = n - 1; i >= 0; i--) {
        output[count[arr[i] - mn] - 1] = arr[i];
        count[arr[i] - mn]--;
    }

    return output;
}
Complexity
Time
O(N + K)

N elements, K is the range of values (max - min + 1). Always linear.

Space
O(N + K)

Count array of size K, plus output array of size N.

Stable?
Yes

Right-to-left placement preserves relative order of equal elements.

In-Place?
No

Requires extra arrays for count and output.
