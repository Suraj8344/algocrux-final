Binary Tree Maximum Path Sum

A path can bend at any node. Each call returns the best straight-down arm, while a global best captures the best bend — O(N) time

The two-value trick

A path may start and end anywhere and can bend at a node (go up-left, through the node, and down-right). The hard part: what a node returns to its parent is not the same as the best path it can form locally.

Return upward: node.val + max(leftGain, rightGain, 0). A parent can only extend one arm, so you hand it a single straight line. Update global best: node.val + leftGain + rightGain, the full arch that bends here. Clamp negative child gains to 0 so they never drag the answer down.

// per node, bottom-up
L = max(gain(left),  0)
R = max(gain(right), 0)
best = max(best, val + L + R) # arch
return val + max(L, R)     # one arm up

C++ — Binary Tree Maximum Path Sum (LC 124) 
class Solution {
  int best = INT_MIN;

  // returns the best straight-down gain starting at node
  int gain(TreeNode* node) {
    if (!node) return 0;
    int L = max(gain(node->left),  0);   // clamp negatives to 0
    int R = max(gain(node->right), 0);
    best = max(best, node->val + L + R);   // arch that bends here
    return node->val + max(L, R);       // hand parent ONE arm
  }
public:
  int maxPathSum(TreeNode* root) {
    gain(root);
    return best;
  }
};
