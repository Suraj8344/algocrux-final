Tree Recursion — Top-Down Approach

Carry information from parent to child through parameters — decide as you descend, no return value needed to compute — O(N) time

What is top-down recursion?

In a top-down traversal you push knowledge down the tree: each call receives the accumulated state of the path so far (a running sum, a max-so-far, the current depth) as a parameter. You update it, then hand the new value to your children.

Contrast with bottom-up, where children return values that the parent combines. Top-down answers questions like "count nodes whose path from the root satisfies X" or "is there a root-to-leaf path summing to target?" — the answer at a leaf depends only on what flowed down to it.

// top-down: state flows DOWN as a param
dfs(node, carry):
  if node == null: return
  carry = update(carry, node.val)
  if isLeaf(node): check(carry)
  dfs(node.left,  carry)
  dfs(node.right, carry)

Time Complexity
O(N)

Each node is visited once; the carry update is O(1) per node.

Space Complexity
O(H)

Recursion stack depth equals the tree height H (O(N) if skewed).
